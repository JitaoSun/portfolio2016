Hypercube is an interface/System for rally racing car, It is designed for improving the driving experience. This system can interact and analyze the outside environment, help drivers master the surrounding environment information. In addition, the system also monitors the vehicle itself, intelligent alerts and reporting necessary information.

UI ANIMATION REEL

Design Guide

* The primary typeface HYPERCUBE is my own design typeface.

Architecture

This interface/system has two parts, the HUD console and the Vehicle Info console. The HUD consoles will be displayed on the windscreen. It can analyze and remind drivers about the environment. The Vehicle Info console will be displayed on a big touch screen. It shows all the information about the vehicle, helps the navigator get the real-time vehicle information and quickly revise the strategy.

HUD Display

There are not too many elements on the HUD screen, in order to allow the driver to concentrate on driving. All the information on the HUD only assists the driver better understand the road conditions, improve the driving experience.

Fluid Layout

The main content area is a large 3D working space. User can use simple gestures to change the camera angle and perspective. For example, in the initial screen, the main area displays the entire car models. When the operator opens with two fingers to zoom the camera, the shell will gradually expand outward and become more transparent. Meanwhile, the data displayed on the panels also change from the entire car overview data to detail parameters of a single or multi component (base on which object is selected or in focus).

All interfaces are fluid. The General panel's layout, scale and contents can be changed according to the central display. As shown above, because the main display area shows the track and environmental information, the originally secondary display area on the below expend upwards to show more information that related to the main display, in order to support user to grasp more detailed data and analysis.

In this interface design, all the info and data are the real content. I do not only design complex looking, but also the usability and applicability. I try to make every part of this interface logical and usable.

Tags

Tag is a major element in this system. Almost all of the operations and data connections work with tag. The principle is very simple, the user selects the object first, then open/connect with a tag, then it will extend more information or operate consoles.

Color is another major element, It is not only used to distinguish different selection, different flow, but also shows the data conversion. This makes the operation of the entire interface to become more intuitive and easy to understand.

Display Layer

As shown left, there are three display layers. This is not a physical level division, but a principle and visual division. All the selected objects or top level messages such as warning display on the top level. All the body content such as data, tags, panels will display on the main level. All the transition will be on the transform level.

Snap is a beautiful, unique app that helps people happy and easy to complete the last step of their driving. It is completely different from all of other parking app and the most important is, Snap better than them.

DESIGN GUIDE

Snap uses simple, understandable icons and beautiful conspicuous typography to present an intuitive user interface. It makes the users easy to read and think, and fully concentrate on their driving, in order to create a safe and comfortable operating environment.
Snap has a lot natural transition animation and gorgeous, prominent colors. All of these can help user to easily remember the interface and user flow, operating without extra thinking.

Mobel App Design Guide

Mobel App Design

There are three different use cases: General viewe, Media team and the Riders (which is the race participant). The example there is the Riders case.

After sign in, is the Ticket/Pass page. On the top, there is a menu that allows users to quickly navigate to other pages/functions.

The main fountions are on the same hierarchy level. so users can also swipe to change pages.

Ghost race can help riders to compare their times with other racers. Lineup page shows where the rider is in to overall lineup. My Team page will show ride's team members, their events, and their ranking.

Users can always click on the logo to go quickly to their Ticket/Pass page.
